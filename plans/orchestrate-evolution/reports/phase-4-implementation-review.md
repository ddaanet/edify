### Minor Issues

1. **SKILL.md: Phase boundary check reads next step file rather than orchestrator plan**
   - Location: `SKILL.md:160-163` (Section 3.5 opening)
   - Problem: The baseline SKILL.md read the next step file header to detect phase changes (`Read the next step entry's phase field`). The new SKILL.md retains "Read the next step entry's phase field" but the orchestrator plan already contains phase fields in each pipe-delimited step entry — no file read needed. However the current text is ambiguous: "Read the next step entry's phase field" could mean the pipe-delimited entry (correct) or the step file (wrong, bloats context per NFR-1).
   - Note: The current text says "Read the next step entry's phase field. If phase changes (or final step):" — "entry" is ambiguous. The orchestrator plan pipe-delimited format has the phase field inline; no step file read is necessary.
   - **Status**: FIXED — clarified to reference orchestrator plan entry, not step file

2. **SKILL.md: `tdd-auditor` agent referenced without prior definition in scope**
   - Location: `SKILL.md:228` — `delegate to tdd-auditor`
   - Problem: Section 6 references `tdd-auditor` as an agent type. This agent is not defined anywhere in the codebase (not generated by prepare-runbook.py, not a baseline template). It was also referenced in the old baseline. The design.md does not mention `tdd-auditor`.
   - Investigation:
     1. Scope OUT: not listed
     2. Design deferral: not found in design.md or outline.md
     3. Codebase patterns: checked `.claude/agents/` — no `tdd-auditor.md` exists
     4. Conclusion: `tdd-auditor` is a vestigial reference carried from the old SKILL.md, not covered by Phase 4 scope
   - Since this was already present in the baseline and is not in Phase 4 scope (design does not mention it), this is OUT-OF-SCOPE for Phase 4 but warrants a flag. The line should be preserved as-is — it pre-existed and is not a Phase 4 regression.
   - **Status**: OUT-OF-SCOPE — pre-existing reference from baseline, not introduced in Phase 4; Phase 4 scope is SKILL.md rewrite against D-1 through D-6

3. **SKILL.md: Section 3.5 checkpoint prompt missing `recall` instruction from baseline**
   - Location: `SKILL.md:170-188`
   - Problem: The baseline checkpoint template included a `**Review recall:**` field with `claudeutils _recall resolve` instruction, which injected plan-specific recall context into the corrector. The new checkpoint prompt drops this entirely, reducing corrector quality for recall-equipped plans.
   - Design check: The design.md does not explicitly list this in scope or out of scope for Phase 4. The outline says the corrector agent has design+outline cached — recall injection was an orchestrator-level concern in the baseline template.
   - Since the new corrector agent has design+outline cached via agent definition (D-2), and the checkpoint prompt calls out `**Design reference:**` only, this is a regression. However the scope says corrector agent behavioral changes are out of scope — the corrector agent content is unchanged; this is the orchestrator's checkpoint *prompt* template. The recall line belonged to the orchestrator prompt, not the corrector definition.
   - Investigation:
     1. Scope OUT: "Corrector agent changes (beyond plan-specific corrector agent generation)" is out of scope, but this is the orchestrator checkpoint prompt, not the corrector agent file
     2. Design deferral: not found
     3. Pattern: Baseline had `**Review recall:** \`Bash: claudeutils _recall resolve plans/<name>/recall-artifact.md\`` in checkpoint template
     4. Conclusion: Fixable — add recall line back to checkpoint prompt template; it was dropped without design justification
   - **Status**: FIXED — restored recall line to checkpoint delegation template

4. **refactor.md: Deslop pass (Step 2b) is between Step 2 and Step 3 — ordering differs from design spec**
   - Location: `refactor.md:101-109`
   - Problem: Design says "add deslop directives before Step 3" and the `Refactoring Evaluation` section's `Factorization before splitting` note is in the Evaluation section (before the protocol steps). Current implementation places `Step 2b: Deslop Pass` as a sub-step between Step 2 and Step 3, and adds `Factorization before splitting` in `Refactoring Evaluation` (line 43). Both placements are correct per the design, but the ordering is slightly off: the design says "before Step 3" which `Step 2b` satisfies.
   - More importantly, the Evaluation section has `**Factorization before splitting:**` at line 43 but the design says this should be in the **Evaluation section** under "Factorization-before-splitting rule". Checking: line 43 is `## Refactoring Evaluation`, and line 43 content is the factorization rule. This matches the design spec exactly.
   - On closer review: this issue resolves to conformant — deslop is Step 2b (before Step 3), factorization is in Evaluation. No fix needed.
   - **Status**: OUT-OF-SCOPE — implementation matches design spec; no issue

5. **refactor.md: Example execution section is slop — narrates behavior already described in protocol**
   - Location: `refactor.md:215-253`
   - Problem: The "Example Execution" section at the bottom of refactor.md is a narration example that restates the protocol (evaluate → tier → execute → return). It adds no information beyond what is in the protocol sections. Code quality rule: no narration comments that restate code in English. The baseline had this same example.
   - The design spec does not require retaining the example. It is slop.
   - **Status**: FIXED — removed Example Execution section

6. **delegation.md: Pre-Delegation Checkpoint still mentions `haiku/sonnet/opus` model check**
   - Location: `delegation.md:18` — "Model matches stated plan (haiku/sonnet/opus)"
   - Problem: The Pre-Delegation Checkpoint verifies "Model matches stated plan (haiku/sonnet/opus)". The delegation.md update removed haiku from Model Selection (correct), but this check still lists haiku as a valid model tier to verify against. If haiku is no longer a valid orchestration model per D-1, the checkpoint should match the updated model tier vocabulary.
   - Design: D-1 says sonnet replaces haiku. The delegation.md Model Selection was updated to remove haiku. The checkpoint should say "(sonnet/opus)" not "(haiku/sonnet/opus)".
   - **Status**: FIXED — updated checkpoint to remove haiku reference
